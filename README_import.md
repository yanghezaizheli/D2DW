# D2DW インポート手順（旧スプシ → Supabase）

`import_d2dw.py` は、既存のGoogleスプレッドシートを Supabase スキーマ用の `seed.sql` に変換するスクリプトです。

## 1. 事前準備

1. Python と依存をインストール: `pip install openpyxl`
2. スプレッドシートを **.xlsx でエクスポート**して1つのフォルダに置く。
   - マスター台帳（OTマップ座標リスト）… 既に .xlsx なのでダウンロードのみ。
   - 区域スプシ（例: 01長住2[OT]）… Googleスプレッドシートで「ファイル → ダウンロード → Microsoft Excel (.xlsx)」。

## 2. 実行（まず dry-run で確認）

```bash
# 抽出サマリ（件数・警告）だけ表示。SQLは書かない
python import_d2dw.py --ledger "OTマップ座標リスト.xlsx" --block "01長住2[OT].xlsx" --dry-run

# 問題なければ seed.sql を生成
python import_d2dw.py --ledger "OTマップ座標リスト.xlsx" --block "01長住2[OT].xlsx" --out seed.sql
```

`--block` は複数指定できます（区域スプシが分かれている場合）。

## 3. Supabase へ適用

SQL Editor で次の順に実行します。

1. `D2DW_supabase_schema.sql`（テーブル・RLS・Realtime）
2. `D2DW_allowlist.sql`（Googleログイン許可リスト）。**ファイル末尾の許可メールを実物に変更してから**実行する。空のままだと全員ログインできません。
3. `seed.sql`（データ投入）。冪等なので、修正して再実行しても二重登録されません。

> Googleログイン: Supabase の Authentication → Providers → Google を有効化し、Google Cloud のOAuthクライアントID/シークレットを設定。リダイレクトURI は `https://<project>.supabase.co/auth/v1/callback`。アプリの公開URL（Vercel等）を Authentication → URL Configuration の Redirect URLs に追加する。

## 4. 取り込みの考え方（重要）

- **訪問ログ方式**: 表示値（前回訪問日時・前回面会日時・累計不在回数）は `visits` から集計します。旧シートの「会えた日付（直近/前回/前々回）」「不在回数」を初期ログに展開します。
- **不在の日時は不明**なので、不在回数ぶんの不在ログを基準日（既定 2026-06-13、`--base-date` で変更可）でまとめて作り、`source='migration'` を付けます。アプリ稼働後の記録は `source='app'` です。
- **ステータス**は `訪問可能/訪問拒否/他言語/JW` 等をそのまま取り込みます（空欄→訪問可能）。
- **除外**: 駐車場・空地・空家・空テナント等は取り込みません。
- **再訪間隔** `visit_rules`（LDR=不在30/投函30/会えた90、EV-M・AL-M=90/30/180 日）を投入します。「訪問可能」かどうかは将来この表で自動判定できます（※ST-Mの値は仮置き＝要確認）。
- **集合住宅→号室**は `parent_id` で接続。建物の座標・種別は台帳から取得します。

## 5. 適用前チェックリスト

- [ ] dry-run のサマリで、区域数・戸数・号室数が現物とおおむね一致するか
- [ ] 建物（集合住宅）の所属区域（block_id）が正しいか ＝ 号室タブの「○○へ」表記で判定。表記が無い建物は既定区域に入るため要確認
- [ ] インライン建物（号室がタブ内に直書き）や「1F 店舗」の扱い（現状は号室として取り込み）
- [ ] 日付が `YYYY-MM-DD` に正しく変換されているか（`seed.sql` を目視）

## 6. 既知の限界（v0.1・要調整ポイント）

- 旧シートはレイアウトが不規則です。列がずれている区域があれば dry-run の警告に出ます。**サマリ出力を共有いただければ列マッピングを調整します。**
- 各戸の地図ピン座標（`map_x/map_y`）は未設定です（設計書の第2段階で、管理画面から登録）。
- 不在の個別日時は近似（基準日まとめ）です。面会日は実日付を保持します。
