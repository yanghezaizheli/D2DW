# D2DW 地図原本ルール

## 目的

D2DW の詳細地図を、Excel 手作業の PNG から、編集しやすい原本管理へ移すための試行ルールです。

当面は `draw.io / diagrams.net` で 1 区域だけ試作し、PNG として書き出して `admin.html` から差し替えます。運用上の限界が見えた場合は、SVG 原本または SVG 表示への移行を検討します。

## 原本形式

- 第一候補: `draw.io / diagrams.net` の `.drawio`
- 書き出し形式: PNG
- 将来候補: SVG
- Excel は当面の参照元として残してよいが、新規修正の正本にはしない

## ファイル命名

原本と書き出し PNG は同じ区域名・同じ番号でそろえます。

例:

```text
maps/source/01nagazumi2-3detail.drawio
assets/images/01nagazumi2-3detail.png
```

命名ルール:

- 区域名は既存PNGに合わせる
- 空白は使わない
- 日本語区域名を使う場合も、アプリ参照用PNGは半角英数字ベースを優先する
- 変更案や一時ファイルは `-draft`, `-test` を付け、本番PNG名にはしない

## キャンバス

ピン座標のズレを防ぐため、同じ区域の地図を差し替える場合は、原則として既存PNGと同じ比率にします。

推奨:

- 既存PNGと同じ幅・高さで作る
- 外枠余白を変えない
- 道路・区画の全体位置を大きく動かさない
- 地図の一部だけを修正する場合も、キャンバス全体のサイズは維持する

既存PNGサイズを確認する方法:

```powershell
Add-Type -AssemblyName System.Drawing
$img = [System.Drawing.Image]::FromFile("assets/images/01nagazumi2-3detail.png")
"$($img.Width) x $($img.Height)"
$img.Dispose()
```

## draw.io 作成ルール

- 道路、区画、建物、文字、目印はできるだけレイヤーを分ける
- 戸番や建物名は後から修正しやすいようテキストとして置く
- 線幅、文字サイズ、色は区域ごとに大きく変えない
- スマホで見る前提なので、細すぎる線や小さすぎる文字を避ける
- 背景は白または薄いグレーを基本にする
- 重要な目印は薄い色だけに頼らず、文字でも分かるようにする

推奨レイヤー:

```text
1_background
2_roads
3_blocks
4_buildings
5_labels
6_notes
```

## PNG 書き出しルール

draw.io から PNG を書き出すときは、次を守ります。

- 背景ありで書き出す
- 透明背景にはしない
- 余白を自動トリミングしない
- 既存PNGと同じキャンバス範囲で出力する
- 画像サイズはできれば 2MB 未満にする

`admin.html` の詳細地図アップロードは PNG を前提にします。

## ピンずれ防止

`places.map_x` / `places.map_y` は、PNG画像上の座標です。PNGのサイズや余白が変わると、既存ピンがずれます。

差し替え前チェック:

- PNGの幅・高さが旧画像と同じか
- 地図の外枠位置が旧画像と同じか
- 代表的な戸や建物の位置が旧画像と同じか
- `admin.html` で既存ピンを表示し、最低5か所を目視確認したか

サイズや余白を変える場合:

- 既存ピンは再配置が必要
- まず1区域だけで試す
- 本番区域全体へ一括展開しない

## 差し替え手順

1. `.drawio` 原本を編集する
2. PNGを書き出す
3. 旧PNGと幅・高さを比較する
4. `admin.html` を開く
5. 対象区域を選ぶ
6. 詳細地図PNGをアップロードする
7. 既存ピンの位置を確認する
8. スマホPWAで表示確認する

## 採用判定

draw.io を正式採用する条件:

- Excelより修正が速い
- PNG差し替え後のピンずれが少ない
- スマホで文字が読みやすい
- 作業者が無理なく編集できる
- 1区域の修正からPNG反映までの手順が安定している

SVG方式を検討する条件:

- PNG差し替えのたびにピンずれが起きる
- 地図上の戸番や目印もDB管理したくなる
- 拡大時の文字や線をもっと鮮明にしたい
- 地図レイヤーをアプリ側でON/OFFしたくなる

## 保管ルール

- `.drawio` 原本は削除しない
- 本番PNGだけでなく、原本も必ず同じタイミングで更新する
- PNGだけを直接編集しない
- 旧版が必要な場合は Git の履歴から戻す

## 試行対象

最初は 1 区域だけで試します。

推奨候補:

```text
assets/images/01nagazumi2-3detail.png
```

理由:

- 既存PNGがある
- admin.html で差し替えとピン確認ができる
- スマホPWAで見え方を確認しやすい
